# Notion 보스 역기획 페이지 — 양산용 양식 기록

**원본 페이지(참고 URL):** [Notion — 보스 역기획 템플릿](https://www.notion.so/3473e5c688d6807e9edcd5bc4cfa6c19)

**자동 파악 한계:** 위 페이지는 **비공개·세션 로그인**이 필요해 외부 도구로 본문 블록을 읽을 수 없었다. 아래는 **저장소 규격(`Document.md`, `docs/reverse_design_phases.md`)과 동일 정보 단위**로 맞춘 **양산 템플릿 스펙**이다. 실제 노션의 블록 순서·제목은 노션에서 **⋯ → Export → Markdown** 으로 뽑아 이 파일의 「실측 블록 순서」절에 붙이면 된다.

## 1. 페이지 역할

| 항목 | 내용 |
|------|------|
| 목적 | 보스 1체(또는 동일 chr 키 묶음)에 대한 **역기획 단일 페이지**를 복제해 채운다. |
| 독자 | 기획·밸런스·문서화 담당 |
| 산출 연동 | `AtkParam` CSV 필터, `data/*_combat.json`, `ListUrl/npc_*.html`, TAE JSON 스키마 |

## 2. 권장 페이지 골격 (블록 타입 가이드)

위에서 아래로 고정 순서를 쓰면 `Document.md` **B형(명세)** + 일부 **A형(카드/콜아웃)** 과 대응하기 쉽다.

### 2.1 표지 블록

| 순서 | Notion 블록 | 채울 내용 |
|------|-------------|-----------|
| 1 | **제목 1** | 보스 표기 — 예: `Morgott (엘데의 왕 모르고트)` |
| 2 | **부제 또는 캡션** | 한 줄 스코프 — 예: `AtkParam 구간 + 자산 인벤토리 + 위키 출처` |
| 3 | **구분선** | (선택) |

### 2.2 요약 대시보드 (B형 상단)

| 순서 | 블록 | 내용 |
|------|------|------|
| 4 | **콜아웃** (info) | `chr` ID, `anibnd`/`chrbnd` 파일명, regulation 버전, 위키 URL 한 줄 |
| 5 | **2열 표** | 열: `데이터 소스` / `역기획서 항목` / `표현 방식` — 아래 「매핑 표」와 동일 논리 |

### 2.3 본문 섹션 (고정 헤딩)

각 **헤딩 2** 아래에 표·토글·이미지를 둔다.

| 헤딩 2 | 권장 내용 |
|--------|-----------|
| **Param (수치)** | `AtkParam_Npc` 발췌 행 링크 또는 임베드 표 이미지; `NpcParam` 요약 행 |
| **TAE / 프레임** | 외부 도구(DS Anim Studio 등) 타임라인 스크린샷 또는 `animations[]` JSON 요약 표 |
| **자산 인벤토리** | `chr/cXXXX.anibnd.dcx` 등 한 줄 목록 (`--asset-doc-lines`와 동일 톤) |
| **위키·출처** | Fextralife 등 **원문 링크** + 「번역·요약」 문구 (`ListUrl` HTML과 동일 정책) |
| **AI / 패턴** (선택) | Lua·거리 분기는 **콜아웃(경고)** 으로 “범위·신뢰도” 명시 |
| **체크리스트** | 초안 완료 시 항목 (`docs/reverse_design_phases.md` 하단 체크리스트 참고) |

## 3. 데이터 소스 ↔ 노션 표현 (매핑 표)

양산 시 **표 한 장**을 페이지 상단에 두고 복제만 하면 된다.

| 데이터 소스 | 기획서 항목 | Notion 표현 방식 |
|-------------|-------------|-------------------|
| `anibnd` (TAE) | 공격 프레임, 선/후딜, 캔슬 타이밍 | 타임라인 스크린샷 / 임베드 / 수식 그래프(범례 고정) |
| `chrbnd` (Model) | 히트박스 위치 참고, 외형 | 이미지 갤러리, Model Editor 스크린샷 |
| `regulation.bin` → Param | 데미지 배율, 강인도, 상태이상 | **데이터베이스** 또는 표 블록 |
| Lua / AI (선택) | 패턴 분기 | 콜아웃 + 순서도(이미지) |

## 4. 데이터베이스로 양산할 때 (선택)

**한 DB = 여러 보스 행**이면 복제가 아니라 **행 추가**로 양산한다.

| 속성 이름 | 타입 | 용도 |
|-----------|------|------|
| `boss_key` | Title | 페이지 제목과 동일 |
| `chr_id` | Text | 예: `c2130` |
| `wiki_url` | URL | Fextralife 원문 |
| `atkparam_note` | Text | CSV 필터 문자열 예: `[Morgott]` |
| `asset_inventory` | Text | 줄바꿈 구분 경로 |
| `tae_status` | Select | `미추출` / `JSON병합완료` 등 |
| `repo_html` | URL | GitHub Pages `npc_*.html` 링크 |

**템플릿 페이지**를 DB의 **템플릿**으로 지정하면 새 행마다 동일 레이아웃이 생긴다.

- [ ] Param 표 또는 CSV 링크 경로
- [ ] 자산 인벤토리 텍스트
