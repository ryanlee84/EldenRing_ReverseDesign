# Smithbox로 보스 전투 역기획 — 범위 가이드

Smithbox는 Elden Ring용 **통합 모딩/데이터 뷰어**다. 공식 README 기준 주요 기능은 맵·모델(FLVER)·파람·텍스트·GPARAM·머티리얼·텍스처·파일 브라우저 등이다.  
참고: [Smithbox README (raw)](https://raw.githubusercontent.com/vawser/Smithbox/main/README.md)

보스 전투를 **역기획**할 때 Smithbox만으로(또는 Smithbox를 중심으로) **어디까지 가능한지**, 그리고 **한계**를 정리한다.

---

## 1. Smithbox로 잘 되는 영역

### 1.1 파람(Param) 중심의 전투 스펙

- `AtkParam_Npc`, `NpcParam`, `SpEffectParam`, 저항·체력·스태미나·탄 관련 파람 등 **숫자·ID·플래그로 정의되는 대부분**은 Param 에디터로 조회·비교·추출하기 좋다.
- 공격별 데미지, 경직, 가드, SpEffect 참조, `hit*_radius` 등 **표 형태 역기획**에 적합하다.

### 1.2 애니메이션 타임라인 이벤트 (TAE / TimeAct)

- Smithbox 계열(DSMapStudio 포크)에는 **TAE(타임 액트)** 조회·편집 기능이 포함되어 있다는 설명이 일반적이다.
- TAE에는 보통 **시간(프레임) 구간**과 함께 **공격 이벤트, 이펙트, 사운드, 이동/판정과 연결되는 참조 ID** 등이 올라간다.
- 따라서 “이 모션의 **몇 프레임부터 몇 프레임까지** 어떤 이벤트가 붙는가” 수준의 **타이밍·연계 단서**는 TAE가 핵심 소스다.
- **게임이 실제로 쓰는 프레임레이트(예: 60fps)와의 대응**은 사용 중인 도구/덤프 방식에 따라 한 번 검증하는 것이 좋다.

### 1.3 리소스 경로·번들 인벤토리

- `*.chrbnd`, `*.anibnd` 등 보스 캐릭터 번들 위치와 파일 묶음은 **파일 브라우저**로 파악하기 쉽다.
- 이 저장소의 전투 문서 파이프라인은 Smithbox 트리 경로를 **자산 인벤토리(경로 나열)** 로만 기록하기도 한다.

---

## 2. 판정·연계까지 역기획할 때의 한계

### 2.1 판정(히트박스)의 기하

- TAE 이벤트와 `AtkParam_Npc`의 `hit*_radius` 등 **파람 필드**로 **일부**는 따라갈 수 있다.
- 그러나 **관절에 붙은 캡슐의 정확한 배치·회전·다단 히트의 3D 형태**까지 “엔진 밖에서 그대로 복제”하려면 **HKX/Havok 애니·별도 뷰어/역공 도구**를 같이 쓰는 경우가 많다.
- 모델 에디터로 **메시를 보더라도** 전투 판정 메시와 1:1이라고 가정하면 안 된다.

### 2.2 기술 연계 = AI·상태 머신

- “왜 이 콤보로 이어지는지”는 **Think / ESD(행동 스크립트) / 이벤트(EMEVD)** 쪽 설명이 필요하다.
- Smithbox에 ESD·이벤트 관련 UI가 있더라도, 커뮤니티 기준으로는 **읽기 위주·미완**으로 언급되는 경우가 있다.
- **파람 + TAE로 역추적 가능한 부분**과 **스크립트까지 완전 역설계**는 난이도와 신뢰도가 다르다.

### 2.3 애니메이션 자체(키프레임 곡선)

- Model 에디터는 **FLVER 등 모델 작업**에 가깝다.
- **DCC용 FBX로 애니 전체를 뽑아 프레임 단위로 편집**하는 워크플로는 보통 **별도 변환 파이프라인**이 필요하다.

---

## 3. “보스 제작용 데이터로 추출 가능한가?”

| 구분 | 내용 |
|------|------|
| **추출·역기획에 가깝다** | 밸런스·이펙트·버프·가드·경직 등 **파람으로 닫히는 스펙**; 모션별 **TAE 이벤트 타임라인**과 **공격 파람 row 연결**; 캐릭터·애니 번들 **파일 구조·이름 규칙** |
| **추가 도구·작업이 필요** | **정밀 히트박스 기하** 재현; **AI/연계 의도**를 설계 문서 수준으로 고정; **애니를 외부 DCC로 재이용** |

정리하면, Smithbox는 **역기획의 허브(파람 + 자산 위치 + TAE 조망)** 로 쓰기 좋다.  
질문에서 말한 **프레임·이벤트·판정 타이밍**은 **TAE + AtkParam 등 파람을 같이 엮는 방식**에서 가능한 범위가 가장 크다.

---

## 4. 이 프로젝트 파이프라인과의 관계

`tools/combat_doc_export.py`는 다음 전제를 둔다.

- `AtkParam_Npc` CSV에서 공격 행을 필터링해 JSON/Markdown을 만든다.
- `.anibnd` / `.chrbnd` 등 **바이너리는 스크립트가 직접 파싱하지 않는다.** `--asset-dir`로 **파일 목록(인벤토리)** 만 문서에 넣는다.
- **TAE 이벤트 표**는 Smithbox UI만이 아니라 **별도 도구로 덤프한 JSON**을 `--merge-tae`로 병합하는 전제다.

즉, 실무에서는 **Smithbox로 조회·추출 + TAE는 전용 덤프/스크립트** 조합이 흔하다.

### Smithbox 2.1.2와 Animation Editor (참고)

현재 공개 소스 기준(태그 `2.1.2`) `SupportsAnimEditor()`가 **항상 `false`**를 반환해, **Elden Ring 프로젝트에서는 Animation Editor(TAE 타임라인 UI)가 동작하지 않도록** 막혀 있다.  
README 등에 “Time Act Editor”가 적혀 있어도 **해당 버전의 ER 워크플로에서는 UI를 기대하지 않는 것**이 안전하다. TAE 내용은 **File Browser로 파일만 꺼낸 뒤** [DS Anim Studio](https://github.com/Meowmaritus/DSAnimStudio) 등 **외부 도구**로 여는 방식을 쓴다.

### File Browser만으로 `anibnd` / `chrbnd` 추출·인벤토리 (에디터 없이)

**목적:** 번들 안에 무엇이 있는지 목록을 확보하고, `.tae`·`.hkx` 등을 **프로젝트 폴더로 추출**해 두었다가 외부 도구·문서 파이프라인에 넘긴다.

1. Smithbox에서 **Elden Ring 프로젝트**를 연다. (게임/데이터 경로가 올바른지 확인)
2. 상단 탭에서 **File Browser**를 연다.
3. 트리에서 **`chr`**(또는 캐릭터 에셋이 모이는 동일 루트)로 이동한다.
4. 검색/필터가 있으면 **`c2130`**(Morgott 본페) 또는 필요한 변형(`c2131` 등)을 찾는다.
5. **`c2130.anibnd.dcx`**(및 **`c2130.chrbnd.dcx`** 등)를 선택하고, UI에 따라 **번들 내부 파일 목록**을 펼친다.
6. 가운데 패널 아래 **Actions** 영역에서 출력 폴더를 정한 뒤,  
   - 번들 **전체**만 빼려면 **「Extract Main File」**,  
   - 목록에서 고른 **내부 파일 하나**(예: `…/tae/c3251.tae`)만 빼려면 **「Extract Selected Internal File」**  
   을 누른다. (Smithbox **2.1.2** File Browser 기준 문구)
7. 추출 후 폴더에서 **파일명 목록**을 한 번 덤프해 두면, 이 저장소의 `combat_doc_export.py`에 넘길 **`--asset-doc-lines`용 텍스트**(`data/*_asset_inventory.txt`)로 바로 쓸 수 있다.

Morgott 전용 단계별 문서는 **`docs/morgott_smithbox_extract.md`** 를 본다. 인벤토리 힌트 템플릿은 **`data/morgott_asset_inventory.txt`** 이다.

## 5. 버전별 확인이 좋은 항목

사용 중인 Smithbox **릴리즈 태그**마다 메뉴가 다를 수 있으므로, 아래는 설치 후 한 번씩 확인하면 좋다.

- [ ] TimeAct / TAE **에디터** 존재 여부(버전·게임 조합별로 비활성일 수 있음) 및 **File Browser 추출**으로 대체 가능한지
- [ ] ESD·EMEVD 등 행동/이벤트 관련 에디터의 **읽기 전용 여부**
- [ ] Param기(CSV 등) 워크플로
